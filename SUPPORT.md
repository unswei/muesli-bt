# support

## where to ask questions

Use GitHub issues for:

- reproducible bugs;
- documentation problems;
- feature requests with a clear use case;
- questions about supported examples or released contracts.

Use the documentation first for:

- [first 10 minutes](docs/getting-started-10min.md);
- [choose your path](docs/getting-oriented/choose-your-path.md);
- [known limitations](docs/known-limitations.md);
- [runtime contract v1](docs/contracts/runtime-contract-v1.md).

## supported surface

The supported surface is defined by release notes, tests, and documentation status labels. Roadmap items are not support commitments until they are implemented, tested, and documented as released.

## security and safety-sensitive reports

Use [SECURITY.md](SECURITY.md) for suspected vulnerabilities, unsafe execution paths, or validation bypasses.
