# v0.5.0 release notes

Release date: 2026-04-09

## what is stable in v0.5.0

- runtime contract and canonical event schema remain unchanged for downstream consumers:

    - `docs/contracts/runtime-contract-v1.md`
    - `docs/contracts/compatibility.md`
    - `schemas/event_log/v1/mbt.evt.v1.schema.json`

- the current flagship baseline remains the shared wheeled BT reused across the supported backend paths:

    - Webots
    - PyBullet
    - ROS2

- the flagship comparison story now has a clearer split between cross-transport and same-robot comparison material:

    - [cross-transport flagship comparison](../examples/cross-transport-flagship-comparison.md)
    - stricter same-robot comparison with the PyBullet e-puck surrogate
    - [shared contract](../integration/cross-transport-shared-contract.md) and [comparison protocol](../integration/cross-transport-comparison-protocol.md) pages

- the newcomer path is now more explicit from the repository root through the first example pages:

    - [Getting Started](../getting-started.md)
    - [What Is muesli-bt](../getting-oriented/what-is-muesli-bt.md)
    - [Brief Lisp Introduction](../lisp-basics.md)
    - [Brief Behaviour Tree Introduction](../bt/intro.md)
    - [Examples Overview](../examples/index.md)

- release artefacts remain split by support surface:

    - generic Ubuntu `x86_64` archive
    - macOS `arm64` archive
    - Ubuntu 22.04 + ROS 2 Humble archive

## compatibility notes

- previous runtime contract version: `1.0.0`
- new runtime contract version: `1.0.0`
- previous event schema: `mbt.evt.v1`
- new event schema: `mbt.evt.v1`
- compatibility impact: additive docs, example, and packaging clarification only

Migration notes for consumers:

- no parser, schema, or runtime-contract migration is required from `v0.4.0`
- tooling consumers can continue to validate canonical event logs with `tools/validate_log.py`
- if you pin a binary artefact, choose the archive that matches your host support surface
- if you are onboarding a new user, start them on the revised docs path from `Getting Started` through `Example: Hello BT`

## what is intentionally limited

- ROS2 support remains the first thin-adaptor baseline:

    - Ubuntu 22.04
    - ROS 2 Humble
    - `Odometry` in, `Twist` out

- live reset remains unsupported; `reset_mode="stub"` remains for tests and harnesses only
- production VLA/provider integration remains host-owned rather than a built-in runtime transport surface
- the release does not broaden the canonical event schema or runtime contract version

## release posture

`v0.5.0` is the flagship-baseline and newcomer-path release.

The runtime semantics stay anchored to the existing contract and event schema.
The main user-facing changes are clearer example routing, more explicit control-loop guidance, and the shared wheeled flagship/comparison story that now reads consistently across the docs.
