# v0.3.1 release notes

Release date: 2026-03-14

## what is stable in v0.3.1

- runtime contract and canonical event schema remain unchanged for downstream consumers:

  - `docs/contracts/runtime-contract-v1.md`
  - `docs/contracts/compatibility.md`
  - `schemas/event_log/v1/mbt.evt.v1.schema.json`
- the released ROS2 thin-adaptor baseline remains:

  - Ubuntu 22.04
  - ROS 2 Humble
  - `(env.attach "ros2")`
  - `nav_msgs/msg/Odometry` input
  - `geometry_msgs/msg/Twist` output
  - `muesli_bt::integration_ros2`
  - `muslisp_ros2`
- release artefacts are now split cleanly by support surface:

  - generic Ubuntu `x86_64` archive (non-ROS)
  - Ubuntu 22.04 `x86_64` archive with ROS 2 Humble support
  - macOS `arm64` archive

## compatibility notes

- previous runtime contract version: `1.0.0`
- new runtime contract version: `1.0.0`
- previous event schema: `mbt.evt.v1`
- new event schema: `mbt.evt.v1`
- compatibility impact: packaging/support clarification only

Migration notes for consumers:

- if you only need the portable runtime, keep using the generic Ubuntu archive
- if you need the ROS2 thin adaptor, use the dedicated Ubuntu 22.04 + Humble ROS archive or build from source on the same baseline
- the ROS-enabled archive requires a matching ROS 2 Humble runtime on the target host; it is not a fully standalone Linux binary distribution
- no parser, schema, or runtime-contract migration is required from `v0.3.0`

## what is intentionally limited

- the ROS-enabled release artefact is still tied to the first supported ROS lane:

  - Ubuntu 22.04
  - ROS 2 Humble
  - `Odometry` in, `Twist` out
- live ROS-backed runs still rely on run-loop artefact JSONL for the current replay/logging flow; direct canonical `mbt.evt.v1` parity for ROS-backed runs remains later work
- live reset remains unsupported; `reset_mode="stub"` remains for tests and harnesses only

## release posture

`v0.3.1` is a packaging and support-posture patch release.

The runtime contract does not move.
The main change is that the GitHub release now publishes a ROS-enabled Ubuntu 22.04 + Humble artefact instead of leaving ROS support as source/install-only from the release page.
