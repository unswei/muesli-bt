# v0.6.0 release notes

Release date: unreleased draft

## what is stable in v0.6.0

- runtime contract and canonical event schema remain unchanged for downstream consumers:

    - `docs/contracts/runtime-contract-v1.md`
    - `docs/contracts/compatibility.md`
    - `schemas/event_log/v1/mbt.evt.v1.schema.json`

- host capability bundle boundaries are now documented:

    - [host capability bundles](../integration/host-capability-bundles.md)
    - [cap.motion.v1](../integration/cap-motion-v1.md)
    - [cap.perception.scene.v1](../integration/cap-perception-scene-v1.md)
    - [host capability BT example](../integration/host-capability-bt-example.md)

- the first minimal capability API path exists for contract smoke coverage:

    - `cap.list`
    - `cap.describe`
    - `cap.call`
    - deterministic `cap.echo.v1` fixture capability

- `planner.plan` request/result documentation now states the v0.6 boundary explicitly:

    - bounded in-runtime action selection stays in `planner.plan`
    - higher-level host services stay behind host capability bundles
    - non-`:ok` planner results still return an action map
    - timeout, no-action, and error paths use the resolved safe action unless a valid bounded candidate exists
    - `planner.v1` logging expectations are documented through the canonical `planner_v1` event type

## compatibility notes

- previous runtime contract version: `1.0.0`
- new runtime contract version: `1.0.0`
- previous event schema: `mbt.evt.v1`
- new event schema: `mbt.evt.v1`
- compatibility impact: additive API, docs, and tests only

Migration notes for consumers:

- existing `v0.5.0` example users do not need to change their BTs
- existing `env.*`, `planner.plan`, ROS2, Webots, and PyBullet entrypoints remain in place
- `cap.call` is available, but only `cap.echo.v1` is implemented as a deterministic fixture capability
- MoveIt, Nav2, detector, and perception adapters are not released in `v0.6.0`

## what is intentionally limited

- `cap.motion.v1` and `cap.perception.scene.v1` are contract pages, not implemented robot adapters
- `cap.echo.v1` is a smoke-test fixture, not a robot-facing feature
- no new ROS2 transport surface is added
- ROS2 `L2` evidence remains focused on the existing thin `Odometry` -> `Twist` lane and canonical artefact validation
- canonical capability call events are deferred until a released capability can affect runtime behaviour

## release posture

`v0.6.0` is a contract and boundary release.

The main user-visible examples remain close to `v0.5.0`.
The important change is that richer future host services now have a documented place to land without turning `env.*` or `planner.plan` into catch-all APIs.
