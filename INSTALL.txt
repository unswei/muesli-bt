muesli-bt install and consumption guide
======================================

This file documents the supported source install flow and downstream package
consumption path for `muesli-bt`.

1) build + install from source
------------------------------

Required tools:
- CMake 3.20+
- Ninja
- C++20 compiler (`clang++` or `g++`)

Runtime-only install (core package boundary):

  cmake -S . -B build/install-core -G Ninja \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_INSTALL_PREFIX="$PWD/build/install-root" \
    -DMUESLI_BT_BUILD_INTEGRATION_PYBULLET=OFF \
    -DMUESLI_BT_BUILD_INTEGRATION_WEBOTS=OFF \
    -DMUESLI_BT_BUILD_PYTHON_BRIDGE=OFF \
    -DMUESLI_BT_BUILD_WEBOTS_EXAMPLES=OFF
  cmake --build build/install-core --parallel
  cmake --install build/install-core

Installed package files include:
- `${prefix}/lib/cmake/muesli_bt/muesli_btConfig.cmake`
- `${prefix}/lib/cmake/muesli_bt/muesli_btTargets.cmake`
- `${prefix}/include/**` public headers
- `${prefix}/share/muesli_bt/contracts/muesli-studio-integration.md`
- `${prefix}/share/muesli_bt/schema/mbt.evt.v1.schema.json`

2) optional integrations
------------------------

PyBullet integration target export:

  -DMUESLI_BT_BUILD_INTEGRATION_PYBULLET=ON

Webots integration target export (requires Webots SDK available, for example
`WEBOTS_HOME=/Applications/Webots.app` on macOS):

  -DMUESLI_BT_BUILD_INTEGRATION_WEBOTS=ON

If Webots SDK is not found, core runtime still builds and installs, and
`muesli_bt::integration_webots` is omitted.

3) consume with find_package
----------------------------

Point CMake to the install prefix:

  -DCMAKE_PREFIX_PATH="$PWD/build/install-root"

Minimal consumer CMake:

  find_package(muesli_bt CONFIG REQUIRED)

  add_executable(app main.cpp)
  target_link_libraries(app PRIVATE muesli_bt::runtime)

Optional integration targets, when exported:

  if(TARGET muesli_bt::integration_pybullet)
    target_link_libraries(app PRIVATE muesli_bt::integration_pybullet)
  endif()

  if(TARGET muesli_bt::integration_webots)
    target_link_libraries(app PRIVATE muesli_bt::integration_webots)
  endif()

4) studio contract assets
-------------------------

`muesli_btConfig.cmake` defines:

  muesli_bt_SHARE_DIR

This points to `${prefix}/share/muesli_bt`, containing the installed Studio
contract and canonical event schema.

Authoritative contract page:
- `docs/contracts/muesli-studio-integration.md`

5) docs and contributor workflows
---------------------------------

For developer setup, test, docs, and formatting workflows, use:
- `docs/getting-started.md`
- `CONTRIBUTING.md`
