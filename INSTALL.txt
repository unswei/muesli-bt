muesli-bt install and consumption
=================================

For the authoritative guide, use:
- `docs/getting-started-consume.md`
- `docs/getting-started.md`

This file is a short summary for source installs and package consumers.

1) build and install runtime
----------------------------

Required tools:
- CMake 3.20+
- Ninja
- C++20 compiler (`clang++` or `g++`)

```bash
cmake -S . -B build/install-core -G Ninja \
  -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_INSTALL_PREFIX="$PWD/build/install-root" \
  -DMUESLI_BT_BUILD_INTEGRATION_PYBULLET=OFF \
  -DMUESLI_BT_BUILD_INTEGRATION_WEBOTS=OFF \
  -DMUESLI_BT_BUILD_INTEGRATION_ROS2=OFF \
  -DMUESLI_BT_BUILD_PYTHON_BRIDGE=OFF \
  -DMUESLI_BT_BUILD_WEBOTS_EXAMPLES=OFF
cmake --build build/install-core --parallel
cmake --install build/install-core
```

2) optional integration targets
-------------------------------

Enable PyBullet integration export:
- `-DMUESLI_BT_BUILD_INTEGRATION_PYBULLET=ON`

Enable Webots integration export:
- `-DMUESLI_BT_BUILD_INTEGRATION_WEBOTS=ON`

If the Webots SDK is not available, the runtime still builds/installs and
`muesli_bt::integration_webots` is omitted.

Enable ROS2 integration export (validated on Ubuntu 22.04 + ROS 2 Humble):
- `-DMUESLI_BT_BUILD_INTEGRATION_ROS2=ON`

When enabled, the package exports `muesli_bt::integration_ros2` and builds the
ROS2-enabled runner `muslisp_ros2`.

3) consume with find_package
----------------------------

Point CMake at the install prefix:
- `-DCMAKE_PREFIX_PATH="$PWD/build/install-root"`

Minimal consumer `CMakeLists.txt`:

```cmake
find_package(muesli_bt CONFIG REQUIRED)

add_executable(app main.cpp)
target_link_libraries(app PRIVATE muesli_bt::runtime)
```

Optional integrations (when exported):

```cmake
if(TARGET muesli_bt::integration_pybullet)
  target_link_libraries(app PRIVATE muesli_bt::integration_pybullet)
endif()

if(TARGET muesli_bt::integration_webots)
  target_link_libraries(app PRIVATE muesli_bt::integration_webots)
endif()

if(TARGET muesli_bt::integration_ros2)
  target_link_libraries(app PRIVATE muesli_bt::integration_ros2)
endif()
```

4) Studio contract and schema
-----------------------------

`muesli_btConfig.cmake` exports:
- `muesli_bt_SHARE_DIR`

The value points to `${prefix}/share/muesli_bt`, which contains:
- contract: `contracts/muesli-studio-integration.md`
- schema: `schema/mbt.evt.v1.schema.json`
